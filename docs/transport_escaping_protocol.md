# 搬運與轉義防護協議（草案 v1）

產出日期：2026-06-13
來源：Ahab 測試網建設實戰（seed/schema 搬運三連損、Claude Code 四發轉義實彈）
定位:行李袋C 候補條款 + CLAUDE.md 修訂素材。總綱:

> **長內容走機器通道(git/CLI),短報文走人手(md5/單行),代碼走檔案(不經 shell),一切落地必驗(checksum)。**

根因認定:搬運與轉義事故的病灶不是通道品質差,是**通道存在本身**——每多一跳人工、多一層引號,就多一個無驗證的變形面。解法按「消滅通道 > 通道機器化 > 載荷不可變形 > 末端驗證」四層遞減,越前治本。

---

## 第一層:消滅通道(治本)

**1-1 Railway CLI 直連(待辦,下次回電腦執行,約半小時)**
- `railway login` 走瀏覽器 OAuth,token 存 CLI 配置,全程不經對話、不經 Claude Code——符合「值由我自己填」紅線。
- 之後 dump 由 Claude Code 在本機 `railway run pg_dump ...` 直落本地檔,Console→剪貼簿→記事本四跳歸零,md5 都可省。

**1-2 git 即通道(已建成,規則化使用)**
- 凡跨工具(web Claude ↔ Claude Code ↔ Lyon)流動的檔案,先問「能不能走 git」:Claude Code commit → Lyon 按 Sync now → web Claude 直讀。
- 能走 git 就禁止貼文字。

## 第二層:通道機器化(人手只碰短報文)

- web Claude 產出檔案 → 一律給**下載連結**,不貼長文(實證:schema.b64 下載通道零損,拖選通道三連壞)。
- Lyon 給 web Claude 內容 → 一律**上傳檔案**,不貼長文(實證:上傳的終端紀錄完好,拖選貼上的損毀)。
- 鐵則:人手只允許搬兩種東西——**檔案本體(點按下載/上傳)** 與 **短可驗報文(md5、單行指令)**。超過一行的內容禁止經剪貼簿。

## 第三層:載荷不可變形(轉義正解)

**3-1 禁 heredoc/-c 長代碼**
- 超過三行的代碼禁止走 `python3 -c` 或 heredoc,一律先落成 .py 檔再執行。腳本檔不經 shell 引號解析,bash↔Python↔Windows 路徑三層轉義面整體消失。副作用全是正收益:可審、可重跑、可 git 追蹤。

**3-2 純 ASCII 落檔法(中文鐵則 v2 升級)**
- 含中文的程式檔由「純 ASCII 生成器腳本」產出,中文一律 `\uXXXX` 逃逸串,註解英文化。
- 原理:載荷全 ASCII 後任何通道物理上無法使其變形;壞了必炸 py_compile,不會靜默變字。
- 實證:write_parser.py → test_parser.py 兩跳全 ASCII,一次成功;此前中文字面量裸奔 heredoc 三次落檔三次返工。

**3-3 中文鐵則適用範圍擴大**
- 原條款只管「改檔案」;實戰證明「執行含中文的臨時腳本」同樣中招(heredoc 被吞 `f:` 事件)。擴為:**任何含中文的代碼,無論寫入或執行,同等管制**。

## 第四層:末端驗證(已立規,補自動化)

**4-1 跨人工通道搬檔必附 checksum(已實證)**
- 源頭 `md5sum`,落地比對,一致才放行。md5 短串(32 字元)是人機協作的最小可信報文。
- 待辦:Claude Code 寫 verify.py(純 ASCII,約十行),輸入預期 md5+路徑,輸出 PASS/FAIL——人只搬短串,比對交機器。

**4-2 驗證結果只收機器輸出原文**
- Claude Code 的轉述通道會吞字(seed 尾行誤報事件);head/tail 轉述不得作為檔案好壞判據。長文本驗證一律降維成 checksum。

## 附:Claude Code 系統性盲區檔案(信任階梯條款素材)

四發實彈,同一病灶——**單層代碼能力正常,shell↔Python↔文本跨層轉換系統性失靈**:

| # | 事故 | 層 |
|---|------|-----|
| 1 | 簡繁同形異碼(U+5220 vs U+522A)憑記憶鍵入路由表 | 文本層 |
| 2 | grep -c 計零回 exit 1,&& 鏈斷頭(期望值為零的檢查) | shell 層 |
| 3 | exec 切片缺 __file__ + 常數不在切片內,雙炸 | Python 層 |
| 4 | '\u2705' 經 bash 雙引號褪層,字元 vs 字面串錯位 | 跨層 |

立法方向:
- 含跳脫序列/heredoc/多層引號的指令**永遠人工過目,不進任何放寬名單**。
- 「宣告與產出不符」(稱全 \uXXXX 實為中文字面量)計違規,扣信任計數。
- 驗證步驟不得由 Claude Code 自行省略(三驗偷工事件);省略檢查的決定權在 Lyon。

## 附:權限通行證三原則(今日判例)

1. **精確匹配可發**:`python3 tests/test_parser.py` 逐字。
2. **範圍受限可發**:`docker exec ahab-test-db *`(爆炸半徑=拋棄容器)。附帶:容器名通行證綁名字,**ahab-test-db 永不復用於非拋棄用途**。
3. **無界通配永不發**:`git *`、`docker run *`、`docker exec *`、`python3 -c *`。
4. 純讀偵察(open+讀+ascii() 打印、無寫入)由 Lyon 自主放行,不回網頁逐發確認。
